# Co-Study — 팀 지침 & 체크리스트

> 디자인씽킹 수업 팀 프로젝트  
> 주제: AI 시대의 올바른 학습 — 진짜로 작동하는 AI 학습 도우미 만들기

---

## 우리가 만들고자 하는 것

> **Our Goal: 진짜 이번/다음 시험에 사용 가능한 학습 Tool 제작**

AI가 답을 주는 게 아니라, **사용자가 먼저 생각을 꺼내도록 유도하는** 학습 도우미.

기존 학습 AI 플랫폼은 정보를 AI → 사용자 방향으로 흘린다.  
우리는 그 방향을 뒤집는다.

---

## 어떻게 진행하나

### 기록 방식
- **노션**: 회의록, 일정, 빠른 메모
- **co-study-ux.vercel.app**: 공감 경험, UX 분석, VPC, 프로토타입 — 포트폴리오로도 남기는 공간

### 프로세스
```
Empathize  →  Our Experience + Service Research
               ↓
VPC        →  (by Empathize Research)
               ↓
구현        →  Figma & Web (Claude Code)
               ↓
자체 테스트, 추가 Research → 고도화
```

---

## 구현에 대한 철학

**코드보다 기능이 중요하다.**

우리 팀에 API 개발 경험자가 없어도 괜찮다.  
어떤 기능이 사용자에게 진짜로 필요한지를 정의하는 것이 훨씬 더 중요한 일이다.

구현은 Claude Code(AI 코딩 도구)를 적극 활용한다.  
우리가 할 일은 **무엇을 만들지 명확히 정의하는 것**.  
처음부터 완성된 걸 만들려 하지 않는다. 동작하는 최소 기능부터.

---

## 팀 운영 원칙

**각자 바쁜 사람이다.**  
너무 많은 걸 부담하지 않되, 맡은 건 최선을 다한다.

**명확하게 요청하고 확실하게 피드백한다.**  
특히 기능을 논할 때 — 애매한 말보다 구체적인 한 문장이 낫다.

**"해보자" 마인드로.**  
완성이 목표가 아니라 배움이 목표다.  
틀려도 되고, 바꿔도 된다.

### Phase 2 — VPC

- [ ] Customer Profile 초안 완성 (Jobs / Pains / Gains)
