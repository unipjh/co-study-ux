# project_goal.md — Co-Study 프로젝트 목표

## 한 줄 목표

> AI 시대에 진짜로 작동하는 학습 도우미를 만든다.

---

## 배경 — 왜 이 문제인가

생성형 AI가 널리 쓰이면서 학생들은 AI로 공부하고 과제를 한다.  
그런데 결과가 들쑥날쑥하다.

같은 AI를 써도 어떤 경우엔 A+, 어떤 경우엔 B+.  
차이는 AI의 성능이 아니었다.

**차이는 내 머릿속에 틀이 있었냐 없었냐였다.**

AI가 답을 주기 전에, 내가 먼저 생각을 꺼낼 수 있었을 때 학습이 일어났다.  
AI가 첫 번째 입력이 됐을 때, 수용 자체가 안 됐다.

기존 학습 AI 플랫폼 대부분은 정보를 AI → 사용자 방향으로 흘린다.  
요약해주고, 풀이해주고, 설명해준다.  
사용자는 소비자가 된다.

---

## 우리가 만들고자 하는 것

**정보의 방향을 뒤집는 학습 도우미.**

사용자가 먼저 설명하고 → AI가 반응하는 구조.  
답을 주는 AI가 아니라, 생각을 꺼내게 만드는 AI.

---

## 프로젝트 진행 방식

디자인씽킹 프로세스를 따르되, 실용적으로 운영한다.

```
Empathize (공감)
├── 팀원 개인의 학습 경험 수집
├── 기존 학습 AI 플랫폼 UX 분석
└── (선택) 인터넷 사용자 NEEDS 수집

↓

VPC (문제정의 + 아이디어를 통합)
├── Customer Profile: Jobs / Pains / Gains
└── Value Proposition: Pain Relievers / Gain Creators / Product

↓

Prototype
├── Figma 플로우 다이어그램 (디자이너)
└── Claude Code MVP (개발)

↓

Test
└── 소규모 실제 사용자 테스트
```

---

## 이 사이트(co-study-ux)의 역할

프로젝트의 모든 리서치와 기록을 아카이빙하는 공간.

- **Empathize: Experience** — 팀원 학습 경험 기록
- **Empathize: Service UX** — 경쟁 플랫폼 UX 분석
- **VPC** — 문제 정의와 가치 제안을 누적 업데이트
- **Prototype** — 결과물 링크

수업 중간보고서의 뼈대가 되고, 포트폴리오로도 남긴다.

---

## 현재 VPC 초안 (업데이트 예정)

### Customer Profile

**Jobs (해야 할 일)**
- 개념을 이해하고 시험에서 설명할 수 있어야 한다
- 배운 것을 내 언어로 재구성할 수 있어야 한다

**Pains (고통)**
- AI 써도 시험 보면 막힌다
- 읽히는데 설명하려 하면 구멍이 드러난다
- 이해했는지 스스로 확인하는 방법을 모른다
- 메타인지가 없으면 AI 튜터에 뭘 물어봐야 할지도 모른다

**Gains (원하는 것)**
- 내 언어로 재구성된 이해
- 설명할 수 있다는 자신감
- 공부한 게 시험에서 실제로 나오는 경험

### Value Proposition

**Pain Relievers**
- 답 주기 전에 먼저 설명하게 유도
- 설명의 구멍을 AI가 짚어줌

**Gain Creators**
- 내 틀이 생기는 경험
- 이해했는지 확인하는 루프

**Products & Services**
- (UX 분석 후 구체화 예정)
