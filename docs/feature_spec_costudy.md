# Co-Study 학습 도우미 — 기능 명세 및 사용자 플로우

> v0.1 · 2026-04-04  
> 디자인씽킹 팀 프로젝트 MVP 구현 참고용

---

## 1. 제품 개요

PDF·이미지·텍스트 자료를 불러와, 하이라이트·메모·AI 인터랙션으로 학습을 돕는 웹 애플리케이션.

**핵심 가치:** 자료를 보면서 곧바로 이해하고, 기록하고, AI에게 물어볼 수 있다.

---

## 2. 기능 목록

| 번호 | 기능명 | 분류 |
|------|--------|------|
| 2-1 | 자료 렌더링 | Core |
| 2-2 | 텍스트 선택 (하이라이트 · 메모) | Core |
| 2-3 | AI 학습 인터랙션 | 차별화 |

## 3. 전체 사용자 플로우

```mermaid
flowchart TD
    A([앱 접속]) --> B[자료 업로드 / 선택]
    B --> C[캔버스에 자료 렌더링]

    C --> D{사용자 행동}

    D -->|스크롤 / 페이지 이동| C
    D -->|텍스트 드래그| E[텍스트 선택 팝업 표시]
    D -->|하이라이트 클릭| F[메모 · AI 패널 열기]
    D -->|AI 추천 알림| G[추천 자료 위치로 이동]

    E --> H{행동 선택}
    H -->|하이라이트| I[색상 선택 → 저장]
    H -->|메모 추가| J[메모 입력 → anchor 저장]
    H -->|AI 요청| K[AI 인터랙션 패널]

    I --> C
    J --> C
    K --> L{AI 기능 선택}
    L -->|요약| M[AI 요약 생성]
    L -->|설명| N[AI 설명 생성]
    L -->|문제 생성| O[AI 문제 생성]
    M & N & O --> P[AI 응답 → 메모로 캔버스 삽입]
    P --> C

    F --> Q[메모 조회 · 수정 · 삭제]
    Q --> C
```

## 4. 기능별 상세 플로우

### 4-1. 자료 렌더링 (2-1)

```mermaid
flowchart LR
    A[자료 선택] --> B{파일 타입?}
    B -->|PDF| C[PDF 파서\npdf.js 등]
    B -->|이미지| D[Image 렌더러]
    B -->|텍스트| E[Text 렌더러]
    C & D & E --> F[통합 캔버스]
    F --> G[줌 컨트롤\n+ / -]
    F --> H[뷰 전환\n페이지 단위 / 스크롤]
    F --> I[프리로딩\n앞뒤 페이지 미리 로드]
```

**상태값**

| 상태 | 설명 | 초기값 |
|------|------|--------|
| `currentPage` | 현재 페이지 번호 | 1 |
| `zoomLevel` | 확대 배율 | 1.0 |
| `viewMode` | `'page'` \| `'scroll'` | `'page'` |
| `isLoading` | 로딩 여부 | false |

### 4-2. 텍스트 선택 · 하이라이트 · 메모 (2-2)

```mermaid
flowchart TD
    A[텍스트 드래그] --> B[Selection 객체 추출\n시작 위치 · 끝 위치 · 텍스트]
    B --> C[팝업 툴바 표시]

    C --> D{사용자 선택}

    D -->|하이라이트| E[색상 선택\n🟡중요 🔵이해필요 🟢암기]
    E --> F[highlight 데이터 저장\nstart · end · color · text]
    F --> G[캔버스 재렌더링\n해당 구간에 색상 오버레이]

    D -->|메모| H[메모 입력창 표시]
    H --> I[텍스트 입력 → 저장]
    I --> J[anchor 아이콘 표시\n원문 위치에 말풍선 핀]
    J --> K[메모 리스트 패널에 추가]

    G --> L{하이라이트 클릭?}
    L -->|클릭| M[연결된 메모 표시\n또는 AI 패널 열기]

    K --> N[전체 메모 패널]
    N --> O[메모 클릭 → 원문 위치로 scroll sync]
```

**중복 하이라이트 처리 규칙**

```
기존 구간과 겹칠 경우:
  - 완전 포함: 새 색상으로 덮어쓰기
  - 부분 겹침: 겹치는 구간은 새 색상 적용, 나머지 유지
  - 동일 구간: 색상만 변경
```

**데이터 구조**

```json
// highlight
{
  "id": "hl_001",
  "pageIndex": 2,
  "startOffset": 140,
  "endOffset": 210,
  "text": "선택된 텍스트 내용",
  "color": "yellow",
  "memoId": "memo_001"   // 연결된 메모 (없으면 null)
}

// memo
{
  "id": "memo_001",
  "highlightId": "hl_001",
  "pageIndex": 2,
  "anchorOffset": 140,
  "content": "메모 내용",
  "createdAt": "2026-04-04T12:00:00Z",
  "type": "manual"       // "manual" | "ai"
}
```

### 4-3. AI 학습 인터랙션 (2-3)

```mermaid
flowchart TD
    A[텍스트 선택 또는 하이라이트 클릭] --> B[AI 패널 열기]
    B --> C{AI 기능 선택}

    C -->|요약| D["[선택 텍스트] 를 3줄로 요약해줘"]
    C -->|설명| E["[선택 텍스트] 를 쉽게 설명해줘"]
    C -->|문제 생성| F["[선택 텍스트] 로 퀴즈 1개 만들어줘"]

    D & E & F --> G[Claude API 호출\n컨텍스트: 선택 텍스트 + 페이지 정보]
    G --> H[AI 응답 스트리밍 표시]
    H --> I{저장?}
    I -->|저장| J[AI 메모로 캔버스 삽입\ntype: 'ai']
    I -->|닫기| K[패널 닫기]

    subgraph 추천 시스템
        L[사용자 행동 로그\n하이라이트 빈도 · 메모 위치]
        L --> M{재방문 추천 조건\n동일 구간 2회 이상 하이라이트\n메모 없는 하이라이트 다수}
        M -->|조건 충족| N[추천 알림 표시\n이 부분 다시 볼까요?]
        N --> O[해당 위치로 이동]
    end
```

**AI 호출 컨텍스트 구조**

```
시스템 프롬프트:
  "학습 도우미입니다. 학생이 선택한 텍스트에 대해 [기능]을 수행하세요.
   응답은 간결하게, 학습에 도움이 되도록."

유저 메시지:
  선택 텍스트: {selectedText}
  페이지: {pageIndex}
  요청: {요약 | 설명 | 문제 생성}
```

## 7. 기술 스택 제안

| 영역 | 라이브러리 | 이유 |
|------|-----------|------|
| PDF 렌더링 | `react-pdf` (pdf.js) | 페이지 단위 렌더링, 텍스트 레이어 지원 |
| 텍스트 선택 | 네이티브 Selection API | 추가 의존성 없음 |
| AI | Anthropic Claude API | 팀 기존 스택 |
| 상태 관리 | Zustand | 가볍고 훅 기반 |
| 데이터 저장 | Firebase Firestore | 팀 기존 스택 |
| 배포 | Vercel | 팀 기존 스택 |
