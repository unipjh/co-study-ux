### 2️⃣ Empathize: Experience — 팀원 학습 경험 기록

> "우리가 AI로 공부했던 경험을 모아두는 곳"

팀원 각자가 AI와 함께 공부했던 경험을 카드로 기록해.  
잘 됐든, 안 됐든 전부 올려.

**카드 1개에 담기는 내용:**

```
작성자
├── 상황    : 어떤 과목 / 어떤 맥락이었나
├── 행동    : AI를 어떻게 썼나
├── 결과    : 잘 됐나, 안 됐나
└── 인사이트 : 왜 그랬을 것 같나
```

**패턴 분석 인사이트** (접고 펼치는 섹션)

22개 경험을 분석해서 뽑은 공통 패턴이 카드 위에 펼쳐져 있어.

| 분야 | 주요 패턴 |
|------|----------|
| 어떻게 AI를 썼나 | 자료 요약·번역이 가장 많음(6건), 개념 설명+문제 반복(5건) |
| 결과 | 55%는 긍정적, 나머지는 부분 성공 or 실패 |
| 왜 차이가 났나 | 프롬프트에 목적이 담겼느냐 / 능동적으로 재정리했느냐 |

> 카드 추가는 **구글 계정으로 로그인한 팀원만** 가능해.

### 4️⃣ VPC — 우리 제품의 가치 정의

> "우리가 만드는 게 사용자가 진짜 원하는 건지 확인하는 곳"

**VPC = Value Proposition Canvas**

두 쪽이 짝을 이루는 구조야:

```
사용자 쪽                         우리 쪽
────────────────────────────────────────────────
Jobs       (해야 할 일)    ↔   Products & Services
Pains      (고통)          ↔   Pain Relievers
Gains      (원하는 것)     ↔   Gain Creators
```

- 항목을 태그처럼 추가·삭제 가능
- 팀원 누구나 실시간으로 추가 가능
- 리서치가 쌓일수록 계속 업데이트하는 문서

**핵심 기능 목록** 버튼을 누르면  
우리가 만들기로 한 기능들을 제목·설명과 함께 볼 수 있어.

## 다음에 만들 것 — Co-Study 학습 도우미

Prototype 탭의 **기능 명세서** 버튼에서 전체 내용 볼 수 있어.  
간단히 요약하면:

```
PDF·이미지·텍스트 자료를 올리면

  ┌────────────────────────────────────────┐
  │  자료가 화면에 렌더링됨 (확대/축소 가능)  │
  └────────────────────────────────────────┘
              ↓ 텍스트 드래그
  ┌─────────────────────────┐
  │  🟡 하이라이트           │ → 색상으로 구분 저장
  │  📝 메모 달기            │ → 원문 위치에 핀으로 표시
  │  🤖 AI에게 물어보기      │ → 요약 / 설명 / 문제 생성
  └─────────────────────────┘
              ↓
  AI 답변이 메모로 캔버스에 삽입됨
```

**기능 3개:**

| 번호 | 이름 | 한 줄 설명 |
|------|------|-----------|
| 2-1 | 자료 렌더링 | PDF, 이미지, 텍스트를 하나의 화면에서 보기 |
| 2-2 | 텍스트 선택 | 드래그 → 하이라이트 + 메모 남기기 |
| 2-3 | AI 인터랙션 | 선택한 텍스트로 AI 요약·설명·문제 즉시 생성 |
