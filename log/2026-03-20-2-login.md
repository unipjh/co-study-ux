# 2026-03-20 — 구글 로그인 팀원 화이트리스트 적용

## 작업 배경

기존 로그인은 "아무 구글 계정"으로나 쓰기 가능한 상태였음.
공개 배포 사이트이므로 팀원이 아닌 외부인도 데이터 추가/삭제 가능한 구조.
→ 팀원 이메일 목록을 관리하고, 목록에 없는 계정은 쓰기 차단하도록 개선.

## 동작 방식

```
구글 로그인 시도
  ↓
이메일이 TEAM_EMAILS 목록에 있음?
  ├── YES → 정상 로그인, 쓰기 권한 활성화
  └── NO  → 즉시 자동 로그아웃 + 헤더에 "팀원 계정이 아닙니다" 표시
```

읽기(조회)는 로그인 여부와 무관하게 모두에게 공개 유지.

## 팀원 추가 방법

`src/lib/teamEmails.js` 파일에서 이메일 추가:

```js
export const TEAM_EMAILS = [
  'unipjh04@gmail.com',
  'teammate2@gmail.com',  // ← 여기에 추가
]
```

이 파일만 수정하면 전체 탭(Experience, ServiceUX, VPC)에 일괄 적용됨.
