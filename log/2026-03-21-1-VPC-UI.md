# 2026-03-21 — VPC 탭 UI 개선: 페어 기반 레이아웃

## 작업 배경

기존 VPC 탭은 Customer Profile / Value Proposition을 단순 2열로 나열.
→ 두 섹션 간의 연결(Jobs↔Products, Pains↔Relievers, Gains↔Creators)이 시각적으로 드러나지 않음.
→ 3열 그리드(왼쪽 | 화살표 | 오른쪽)로 재구성해 같은 행에 페어가 나란히 배치되도록 개선.

## 주요 변경 사항

### 데이터 구조

`SECTIONS` (그룹 기반) → `PAIRS` (페어 기반) 배열로 교체.

```js
const PAIRS = [
  { left: jobs,   right: products,  color: '#4a7c9e' },  // 파랑
  { left: pains,  right: relievers, color: '#9e4a4a' },  // 빨강
  { left: gains,  right: creators,  color: '#4a9e6a' },  // 초록
]
```

### 레이아웃

```
[Customer Profile]    [ ]    [Value Proposition]
─────────────────────────────────────────────────
[Jobs              ] [↔]    [Products & Services]   파랑
[Pains             ] [↔]    [Pain Relievers     ]   빨강
[Gains             ] [↔]    [Gain Creators      ]   초록
```

- `gridTemplateColumns: '1fr 48px 1fr'`
- 각 행이 하나의 페어를 표현
- 중간 열에 `↔` 기호로 연결 명시

### 색상 통일

- Products & Services: 기존 보라(`#7a4a9e`) → Jobs와 동일한 파랑(`#4a7c9e`)으로 통일
- 페어 내 양쪽 카드가 동일 색상으로 "연결" 표현

### 컴포넌트 분리

- `SectionCard` 컴포넌트 추출 — 좌우 카드 동일 컴포넌트 재사용
